Smart Laundry Room

An IoT-based monitoring and reservation system for shared laundry facilities. Using non-invasive sensors and a Raspberry Pi, it transforms standard machines into smart appliances with real-time availability and queue management.

IoT Raspberry Pi Flask Sensors Senior Capstone

Problem

Shared laundry facilities in dorms and apartments operate as unobservable resource networks — users have to physically walk down to check if machines are free. This system transforms that chaotic environment into a visible, managed network with real-time status and fair queue-based scheduling.

Key Features

Real-Time Monitoring

Non-invasive current sensors detect machine states (idle, running, complete) without any modifications to the appliances.

Smart Reservations

Queue-based scheduling with auto-cancellation for no-shows and notifications when your turn arrives.

Multi-Sensor Data

Temperature, humidity, vibration, and airflow sensors provide cycle details and early maintenance alerts.

Web Dashboard

Color-coded machine availability (green/orange/red) with countdown timers, accessible from any device.

Smart Notifications

Customizable alerts via ping, SMS, or email for cycle completion and reservation updates.

Admin Portal

Facility management interface with usage analytics, maintenance controls, and cycle time customization.

Hardware & Tech Stack

  • Raspberry Pi Zero 2 W — central processing unit
  • SCT-013-000 non-invasive clamp-on current sensors
  • ADS1115 16-bit ADC for precise analog-to-digital conversion
  • DS18B20 temperature probes + SHT31-D humidity sensors
  • ADXL345 accelerometer for spin cycle vibration detection
  • Flask web app with responsive mobile-first design
  • RESTful API for real-time sensor data and user interactions
  • Local storage with 7-day retention — works fully offline

Key Innovations

  • Zero machine modifications — fully non-invasive installation
  • Multi-sensor fusion for accurate cycle state detection
  • Intelligent queue management with auto-cancellation for no-shows
  • Predictive maintenance alerts based on usage patterns
  • Peak usage analysis and optimal laundry time recommendations
  • Local-first architecture — no cloud dependency, privacy protected
